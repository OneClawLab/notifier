# notifier - task scheduler daemon & CLI command

A Linux command that works both as a regular CLI (run-and-exit) and as a long-running daemon via `start`/`stop` subcommands. As CLI it manages instant tasks and cron timers; as daemon it watches for file changes and time signals to execute preset shell commands.

## 决策记录

1. **文件即任务**：每个任务是一个独立的 env 格式文本文件（`KEY=VALUE`），文件名即任务标识。daemon 通过监听目录变动发现新任务，处理后移动文件（pending → done/error），天然幂等。
2. **双模式运行**：CLI 子命令负责任务文件的 CRUD，daemon 负责监听和执行。两者通过文件系统解耦，CLI 进程不写 daemon 日志。
3. **CRON 调度内置**：定时任务使用标准 5 字段 CRON 表达式，daemon 内部维护 Job Table，支持动态重载（监控 `timers/` 目录变动，无需重启）。
4. **单实例保证**：通过 PID 文件保证同一时间只有一个 daemon 实例运行。

## 1. Role

- **As CLI**: Manage instant tasks and cron timers (add / list / remove).
- **As Daemon**: Watch for file changes and time signals, execute preset shell commands.

## 3. Data Directory Layout

Base path: `~/.local/share/notifier/` (override via `NOTIFIER_HOME` env var).

```
~/.local/share/notifier/
├── tasks/
│   ├── pending/    # Instant tasks awaiting execution
│   ├── done/       # Successfully executed tasks
│   └── error/      # Tasks that failed validation
├── timers/         # Cron timer files
└── logs/           # Daemon runtime logs
```

## 4. Data Protocol

所有任务文件使用 env 格式（`KEY=VALUE`，每行一个），后缀统一为 `.txt`。每个文件只包含一个任务。

### 4.1 Instant Task File (`tasks/pending/*.txt`)

Required fields:

| Field | Description |
|-------|-------------|
| `author` | Creator identifier, used to distinguish different authors |
| `task_id` | Task ID provided by the author; notifier does not interpret its meaning |
| `command` | Shell command to execute (single line, may contain pipes/composition) |

Auto-generated fields:

| Field | Description |
|-------|-------------|
| `created_at` | Task creation time, ISO 8601 (written by `task add`) |

Optional fields:

| Field | Description |
|-------|-------------|
| `description` | Human-readable task description, provided by caller |

Example:
```
author=agent-007
task_id=build-42
command=cd /opt/app && make build 2>&1 | tee /tmp/build.log
description=Build the app and capture output
created_at=2026-03-18T10:30:00Z
```

文件名约定：`<author>-<task_id>.txt`（由 `task add` 子命令自动生成）。

### 4.2 Timer File (`timers/*.txt`)

Required fields:

| Field | Description |
|-------|-------------|
| `author` | Creator identifier |
| `task_id` | Task ID provided by the author |
| `command` | Shell command to execute (single line) |
| `timer` | CRON expression (standard 5-field: min hour day month weekday) |

Auto-generated fields:

| Field | Description |
|-------|-------------|
| `timer_desc` | Human-readable CRON description in English (auto-generated by notifier) |
| `created_at` | Task creation time, ISO 8601 (written by `timer add`) |

Optional fields:

| Field | Description |
|-------|-------------|
| `description` | Human-readable task description, provided by caller |

Example:
```
author=ops
task_id=cleanup-logs
command=find /var/log/app -name '*.log' -mtime +7 -delete
timer=0 2 * * *
timer_desc=Every day at 2:00 AM
description=Clean up application logs older than 7 days
created_at=2026-03-18T10:30:00Z
```

文件名约定：`<author>-<task_id>.txt`。

## 5. CLI Commands

遵循 ProgressiveDiscovery.md 的渐进披露规范。所有命令和子命令支持 `--help`、`--help --verbose`、`--version`。无参数或参数错误时自动显示 help（退出码 2）。

### 5.1 `notifier task`

Manage instant tasks.

#### `notifier task add`

Add an instant task file to `tasks/pending/`.

**Args**:
- `--author <name>` (required)
- `--task-id <id>` (required)
- `--command <cmd>` (required)
- `--description <text>` (optional)

**Behavior**:
- 自动写入 `created_at` 字段（当前时间，ISO 8601）。
- 生成文件 `tasks/pending/<author>-<task_id>.txt`。
- 如果同名文件已存在，报错退出（退出码 1）。

Supports stdin: 如果未提供 `--command`，从 stdin 读取命令（单行）。

#### `notifier task list`

List instant tasks.

**Args**:
- `--status <pending|done|error>` (optional, default `pending`)
- `--json` (optional)

**Behavior**: 列出指定状态子目录下的所有任务文件，输出摘要信息（author、task_id、command 截断显示）。

#### `notifier task remove`

Remove an instant task file.

**Args**:
- `--author <name>` (required)
- `--task-id <id>` (required)
- `--status <pending|done|error>` (optional, default `pending`)

**Behavior**: 删除对应文件。文件不存在时报错退出（退出码 1）。

### 5.2 `notifier timer`

Manage cron timers.

#### `notifier timer add`

Add a timer file to `timers/`.

**Args**:
- `--author <name>` (required)
- `--task-id <id>` (required)
- `--command <cmd>` (required)
- `--timer <cron-expr>` (required — standard 5-field CRON)
- `--description <text>` (optional)

**Behavior**:
- 自动生成 `timer_desc` 字段（CRON 的英文可读描述）。
- 自动写入 `created_at` 字段（当前时间，ISO 8601）。
- 生成文件 `timers/<author>-<task_id>.txt`。
- 如果同名文件已存在，报错退出（退出码 1）。
- CRON 表达式校验失败时报错退出（退出码 2）。

#### `notifier timer list`

List all timers.

**Args**:
- `--json` (optional)

**Behavior**: 列出 `timers/` 下所有任务文件，输出摘要信息（author、task_id、timer、timer_desc、command 截断显示）。

#### `notifier timer remove`

Remove a timer file.

**Args**:
- `--author <name>` (required)
- `--task-id <id>` (required)

**Behavior**: 删除对应文件。文件不存在时报错退出（退出码 1）。

### 5.3 `notifier status`

Show daemon running status.

```bash
notifier status [--json]
```

- 读取 `$NOTIFIER_HOME/notifier.pid` 文件，检查对应进程是否存在。
- 输出 daemon 是否正在运行（含 PID）。
- `--json` 输出：`{ "running": boolean, "pid": number | null }`
- 无论 daemon 是否运行，均以退出码 0 退出。

### 5.4 `notifier start`

Start the daemon process.

```bash
notifier start [--foreground]
```

- 默认以后台 daemon 方式运行（spawn detached 子进程）。
- `--foreground`：前台运行，日志同时输出到 stdout 和日志文件，适合调试。
- 同一时间只允许一个实例运行，已有实例时报错退出（退出码 1）。

### 5.5 `notifier stop`

Stop the running daemon.

```bash
notifier stop
```

- 读取 PID 文件，向 daemon 发送 SIGTERM。
- 等待最多 5 秒确认进程退出，超时则提示手动 kill。
- daemon 未运行时报错退出（退出码 1）。

### 5.6 `notifier --help`

显示主命令帮助，列出所有子命令及简要说明。遵循渐进披露规范（约 50 行以内），末尾提示 `--help --verbose` 可获取完整信息。

### 5.7 `notifier --version`

Print version number.

## 6. Daemon Mode

通过 `notifier start` 启动（默认后台运行），或 `notifier start --foreground` 前台运行。

### 6.1 Instant Task Watching

- 监听目录：`$NOTIFIER_HOME/tasks/pending/`
- 监听方式：使用 `fs.watch`（Node.js 内置，跨平台）或 `inotify`（Linux）监听文件创建/移入事件。
- 触发动作：
  1. 读取新文件内容，解析 env 格式。
  2. 格式校验失败：向 stderr 打一行警告日志，立即将文件移动到 `tasks/error/`。
  3. 格式校验成功：执行 `command` 字段中的 Shell 命令。
  4. 执行完成后（无论命令退出码），将文件移动到 `tasks/done/`。
  5. 如果命令返回非零退出码，记录错误日志到 stderr，但 daemon 不崩溃。

### 6.2 Cron Timer Scheduling

- 配置目录：`$NOTIFIER_HOME/timers/`
- 启动时扫描 `timers/` 目录，解析所有定时任务文件，构建内存中的 Job Table。
- 动态重载：监控 `timers/` 目录变动，文件增删改时重新解析并更新 Job Table，无需重启。
- 定时任务文件格式校验失败时：不移动源文件，仅向 stderr 打错误日志。
- 到达 CRON 触发时间时执行对应命令，并计算下一次执行时间。

### 6.3 Main Loop

使用非阻塞 IO 驱动：

1. **Initialize**：扫描 `timers/` 构建 Job Table；扫描 `tasks/pending/` 处理残留文件。
2. **Calculate Sleep**：找到最近的 CRON 触发时间，计算等待时长 T。
3. **Poll**：等待以下事件之一：
   - `tasks/pending/` 有文件变动。
   - 达到超时时间 T（CRON 到期）。
   - 收到终止信号（SIGTERM / SIGINT）。
4. **Execute**：根据事件类型执行对应任务。
5. **Repeat**：回到步骤 2。

### 6.4 Concurrency & Robustness

- 串行处理 `tasks/pending/` 中的任务文件，保证调度顺序。
- 命令执行的并行性由被调用命令自行决定。
- 收到 SIGTERM/SIGINT 时优雅退出（等待当前任务完成）。

## 7. Output Format

### 7.1 stdout / stderr Contract

- `stdout`: Command result data (task list, timer list, etc.).
- `stderr`: Progress, debug, error, and warning messages.

### 7.2 Human / Machine Readability

- Default output is human-readable.
- `--json` enables structured JSON output (`task list`, `timer list`).

## 8. Error Handling & Exit Codes

### 8.1 Exit Codes

| Code | Meaning |
|------|---------|
| `0` | Success |
| `1` | Logic error (file not found, duplicate add, etc.) |
| `2` | Usage/argument error (missing args, invalid CRON, auto --help trigger) |

### 8.2 Error Output

- Default (no `--json`): human-readable error to `stderr`, format `Error: <what went wrong>. <how to fix>`.
- `--json` mode: `{"error": "...", "suggestion": "..."}`

Example: `Error: task file already exists at ~/.local/share/notifier/tasks/pending/agent-build-42.txt. Use a different --task-id or remove the existing task first.`

## 9. Logging

### 9.1 CLI Mode

错误和警告输出到 stderr。CLI 进程不写 daemon 的日志文件（不同进程）。

### 9.2 Daemon Mode

Log file: `$NOTIFIER_HOME/logs/notifier.log`

**记录内容**（覆盖调度各环节，便于追查调度历史和故障排查）：
- daemon 启动/停止事件
- `tasks/pending/` 文件变动检测
- 即时任务文件解析结果（成功/失败及原因）
- 即时任务命令执行开始、结束、退出码、耗时
- 即时任务文件移动（→ done 或 → error）
- `timers/` 目录变动检测与 Job Table 重建
- 定时任务文件解析结果（成功/失败及原因）
- 定时任务 CRON 触发、命令执行开始、结束、退出码、耗时
- 下一次 CRON 触发时间计算结果
- 信号处理（SIGTERM/SIGINT 收到、优雅退出过程）

**Log line format**:
```
[2026-03-18T10:30:00.123Z] [INFO] daemon started, NOTIFIER_HOME=~/.local/share/notifier
[2026-03-18T10:30:01.456Z] [INFO] task detected: agent-007-build-42.txt
[2026-03-18T10:30:01.500Z] [INFO] task executing: author=agent-007 task_id=build-42 command="cd /opt/app && make build"
[2026-03-18T10:30:05.789Z] [INFO] task completed: author=agent-007 task_id=build-42 exit_code=0 duration=4289ms → done
[2026-03-18T10:30:06.000Z] [WARN] task parse failed: bad-file.txt reason="missing required field: command" → error
[2026-03-18T10:31:00.000Z] [INFO] cron triggered: author=ops task_id=cleanup-logs timer="0 2 * * *"
```

### 9.3 Rotation Policy

- 触发条件：daemon 每次重启时，或当前日志文件超过 10000 行时。
- 轮换方式：将当前 `notifier.log` 重命名为 `notifier-<YYYYMMDD-HHmmss>.log`，然后创建新文件。

## 10. Idempotency & Safety

1. **task add / timer add**：不覆盖已存在的同名文件（非幂等，重复添加报错）。
2. **task remove / timer remove**：删除不存在的文件报错（幂等性不适用）。
3. **Daemon task processing**：同一文件只会被处理一次（处理后移走），天然幂等。
